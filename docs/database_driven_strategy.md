# Database-Driven Strategy

Use markdown for authorship and PostgreSQL for indexing.

## Best long-term combination

- markdown content files for drafting
- Postgres for relationships and infinite story growth
- Census + BLS ETL jobs for data snapshots
- optional Google Civics and Maps integrations for lookup and visualization

## Why this matters

Once stories, demographics, ballot fights, and course components are relationally stored,
you can spawn new course tracks almost indefinitely from the same system.
